# v0.2.2 — 2026-07-15

## Added

- `dfw_traffic`: four sources behind one `kind` enum — Fort Worth real-time
  accidents (`incidents`), Dallas right-of-way-permit street/lane closures
  (`closures`), TxDOT 5-Year AADT counts (`counts`), TxDOT construction
  projects (`projects`). `kind=all` merges only the two "happening now"
  sources; counts/projects need an explicit kind.

## Why it's shaped this way (source archaeology)

- The obvious Dallas ROW Socrata IDs (`yi5a-ym5z` lines / `xum9-x6px`
  points) are **dead shells** — map-visualization views with zero columns.
  The real datasets (`xd3q-ipis` / `bw6g-a3ur`) were found via each shell's
  `modifyingViewUid`. Recency uses `createddate`; `issuedate` carries
  placeholder/future dates.
- TxDOT county fields store Title Case ("Dallas", not "DALLAS"); the ArcGIS
  backend happens to match case-insensitively, but nothing here relies on
  that accident.
- The AADT current-year field is frequently null while older history fields
  have data: the tool picks the first non-null across the five fields and
  labels it with that row's own `LATEST_AADT_YR`.
- The AADT layer has **no road-name field** — `search` is ignored for
  `kind=counts`, with a note saying so.
- Pagination is a top-K merge per sub-source; `total_matched` is an honest
  lower bound (shown as "N+" while more pages exist), not a full count.
